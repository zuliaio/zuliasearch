## Breaking Changes

- `DataSourceRecord.getDouble(String, int)` has been renamed to `getInt(String, int)`. The previous name was incorrect and the method has always returned an `int`. Update any callers that were relying on the misnamed method.

## New Features

- **Learned sparse encoder support.** Zulia now supports learned sparse embedding models alongside dense vector search
- **BIOCLINICAL_MODERNBERT embedding model.** New entry in `KnownEmbeddingModel` for biomedical and clinical text, using a 768-dimension ModernBERT architecture. Hosted by zuliaio.

## Improvements

- Detailed `displayIndexes` CLI output now includes per-index weight.
- `displayIndexes` CLI is now sorted by case-insensitive A-Z sort index name sort (was index creation order before)
- Nodes REST skips nodes where both the primary and replica shard counts are empty
- `ExcelSourceConfig.withStrictHeaders()` now rejects correctly blank and duplicate header names, matching the behavior of `DelimitedSourceConfig`.

## Bug Fixes

- `DisplayIndexCmd` now requires the `--view` argument, preventing a NullPointerException when it was omitted.
- `JsonArraySource` iterator no longer silently swallows `IOException` failures

## Internal

- **Maven Central publishing.** Zulia now publishes to the Maven Central Portal
- **Release automation.** A new GitHub Actions workflow (`.github/workflows/release.yml`) builds, signs, and publishes to Maven Central on tag push, and creates a GitHub Release with the server and tools distribution tars attached. Release notes for each version live in `release-notes/<tag>.md` and are read by CI as the release body.
