## Breaking Changes

- **`MasterSlaveSettings` renamed to `PrimaryReplicaSettings`.** In the Java client, `Search.getMasterSlaveSettings()` / `Search.setMasterSlaveSettings(...)` are now `getPrimaryReplicaSettings()` / `setPrimaryReplicaSettings(...)`, and the enum class `MasterSlaveSettings` is now `PrimaryReplicaSettings`. The enum values keep backward-compatible aliases (`MASTER_ONLY`, `SLAVE_ONLY`, `MASTER_IF_AVAILABLE` still resolve to the same ordinals), so the gRPC wire format and stored data are unaffected - only Java source that references the old class or method names needs updating.

## New Features

- **Lucene segment replication.** Primary nodes now replicate committed Lucene segments to their configured replica nodes. Replication uses a per-shard generation gate, a virtual-thread executor, a per-replica circuit breaker, and a shared bandwidth rate limiter (default 60 MiB/s). A 30s watchdog recovers from missed checkpoints. Replicas advertise their Lucene version so a primary rejects mismatches before any transfer, requiring `replicaVersion >= primaryVersion` so operators can roll Lucene upgrades replica-first. A new `/replication/{indexName}` REST endpoint reports per-shard and per-replica progress (last attempted generation, lag, last-success timestamp, circuit-breaker state). New server config: `replicaResponseTimeout` (minutes, default 10) and `replicationMaxBytesPerSec` (default 60 MiB/s, `0` disables the cap).
- **Change the replica count on a live index.** The number of replicas is no longer fixed at index creation and can grow or shrink it on an existing index via `UpdateIndex` (or by re-applying a changed config through `CreateIndex`). Previously this threw "Cannot change replication factor for existing index yet".
- **Configurable, live NRT cache sizes.** The near-real-time RAM cache for an index's index and taxonomy directories is now tunable per index: `nrtIndexMaxMergeSizeMB` (default 50), `nrtIndexMaxCachedMB` (default 150), `nrtTaxoMaxMergeSizeMB` (default 5), `nrtTaxoMaxCachedMB` (default 15). Caching can also be toggled off live via `nrtCachingDisabled`. All of these can be changed through `UpdateIndex` without a restart.
- **Multi-index aliases with a designated write index.** A single `IndexAlias` can now point at multiple indexes and fan queries out across all of them. An optional `writeIndex` designates which member receives single-index mutations (store, delete, batchDelete, clear, optimize, reindex, associated-doc writes, updateIndex, getIndexSettings), enabling the rollover / zero-downtime reindex pattern without juggling two aliases.
- **Hybrid retrieval: per-clause boost and `VECTOR_SHOULD`.** Query clauses now support a `boost` score multiplier (applied via Lucene `BoostQuery`; `0` is treated as unset, `1.0` as a no-op, and it is ignored for non-scoring clause types). The new `VECTOR_SHOULD` query type enables linear hybrid retrieval (BM25 + KNN): the vector clause contributes score but does not constrain the result set to the HNSW top-K.
- **Wildcard index patterns in query requests.** Queries can now target indexes by wildcard pattern instead of naming each index explicitly.

## Improvements

- **Gzip support in `zuliadump` and `zuliarestore`.** Both tools now read and write compressed (gzipped) files.
- **Lower thread overhead at high index counts.** Each index's three maintenance Timers (commit, warm, replication watchdog) are now virtual threads instead of dedicated platform threads, eliminating 3xN platform threads and their idle wakeups when running many indexes. A failing maintenance task no longer kills its loop, and shutdown wakes the loops cleanly instead of interrupting in-flight Lucene I/O.

## Bug Fixes

- **Cleaner shard shutdown.** `ShardWriteManager` now uses `IndexWriter.rollback()` instead of `close()` when shutting down. Because `commit()` already runs earlier in the shutdown path, `rollback()` skips merges and releases the writer cleanly.
- **Reliable gRPC port release on shutdown.** Node shutdown now escalates from a graceful gRPC shutdown to a forced one when in-flight RPCs do not drain in time. Long-lived segment-replication streams could otherwise outlast the grace period and leave the service port bound, so a fast node restart failed to bind.
## Dependencies

- Micronaut 4.10.10 -> 4.10.14 (Micronaut Core 4.10.23), which pulls in Netty 4.2.13 addressing over a dozen CVEs.
