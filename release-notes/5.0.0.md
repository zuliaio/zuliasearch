## Breaking Changes

- **Java 25 is now required.** Published bytecode now targets Java 25 (`source`/`targetCompatibility` moved from 21 to 25), and the build requires JDK 25. Any project that depends on the Zulia libraries (`zulia-client`, etc.) must also compile and run on Java 25.
- **Micronaut 5 / Jackson 3 REST stack.** The server and UI REST applications moved from Micronaut 4.10.x to Micronaut 5.0.0, which switches the JSON engine to Jackson 3 (`tools.jackson.core`). The HTTP and gRPC wire APIs are unchanged,
- **Associated-document upload field order.** `ZuliaRESTClient.storeAssociated` now sends the `metaJson` form field before the `file` part (and always sends it, empty JSON when absent), so Micronaut 5 can bind the metadata alongside the streamed file. Any client posting to the associated-document endpoint should send `metaJson` ahead of `file`. The bundled client now requires Unirest 4.10+ multipart insertion-order behavior.

## Improvements

- **Faster document collection.** The scored-doc hot path (`DocumentScoredDocLeafHandler` / `ScoredDocLeafHandler`) reduces per-document copies and allocations, and drops a dead sort and int boxing.
- **Larger default GridFS chunk size.** Associated documents stored in MongoDB GridFS now use the GridFS default chunk size (255 KB) instead of a forced 1 KB, cutting chunk counts and round-trips for stored files. Existing files keep their original chunk size; only newly stored files are affected.
- **Document filter helpers in `ZuliaUtil`.** Document-filtering logic was consolidated into `ZuliaUtil` for reuse by clients.

## Bug Fixes

- **Correct stats ordering for large longs.** `LongStats.compareTo` now uses `Long.compare`, fixing precision loss that could mis-order very large long values in aggregation sorts.
- **Up-to-date cluster routing.** `ZuliaIndexManager.currentOtherNodesActive` is now `volatile`, so request threads see current cluster membership and route correctly after a membership change instead of using a possible stale view.
- **Reliable commit and warming scheduling.** `ShardWriteManager.lastCommit` / `lastChange` / `lastWarm` are now `volatile`, so the warming / idle-commit scheduler no longer reads a stale or null value and skips or delays needed commits and warming.
- **Atomic single-node heartbeat updates.** `SingleNodeService` keeps its node in an `AtomicReference` updated via `updateAndGet`, giving atomic, cross-thread-visible heartbeat and version updates.

## Data

- `zulia-data` migrated to Jackson 3 (`JsonArraySource` updated for the new parser/mapper API), with a new `JsonArraySourceTest` covering multi-object and empty-array iteration and an iteration bug fix for empty arrays.

## Internal
- Server tests manage embedded MongoDB through a JUnit `LauncherSessionListener`, removing `InterruptedAtShutdown` noise at the end of test runs (matching the ui-rest setup).
