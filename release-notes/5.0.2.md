## Breaking Changes

- **`DataInputStream` now requires `openRawInputStream()` (zulia-data API).** The `io.zulia.data.input.DataInputStream` interface now declares `openRawInputStream()` for the underlying stream and provides `openInputStream()` as a buffered `default` method. Code that *calls* `openInputStream()` is unaffected, but any custom *implementation* of the interface must move its stream logic into `openRawInputStream()`.

## Bug Fixes

- **Large whole-number Excel cells are no longer truncated.** Reading a numeric Excel cell as a string previously truncated whole numbers above `Integer.MAX_VALUE` to "2,147,483,647", silently corrupting large values.  Reading such a cell as an int with `getInt` now throws `ArithmeticException` instead of silently wrapping.
- **Excel sources with an empty sheet no longer crash.** Opening a workbook whose selected (or active) sheet is empty or has a blank first row previously threw a `NullPointerException`. Empty sheets now report zero rows, and switching to an unknown sheet name throws a clear `IllegalArgumentException` instead.
- **CSV/TSV re-iteration no longer leaks file handles.** Re-iterating a delimited source (including header-only files) left the previous reader and its underlying file handle open. `reset()` now closes the prior reader before reopening, and `close()` is idempotent.
- **Empty delimited file with headers gives a clear error.** Opening an empty CSV/TSV with headers enabled now throws a descriptive `IOException` instead of an unchecked `NoSuchElementException`.
- **Excel exports no longer leak temp files.** `ExcelTarget` now disposes the streaming `SXSSFWorkbook` on `close()`. Previously the temporary files POI spills to disk were orphaned under the system temp directory on every export.

## Improvements

- **Consistent stream buffering across data sources.** `DataInputStream.openInputStream()` now always returns a buffered stream regardless of the implementation, with the buffering logic in one place. Gzip-backed file sources read the compressed input in 64KB chunks instead of the 512-byte default.
- **`SingleUseDataInputStream.from(byte[], filename)`** convenience factory added for building a single-use source directly from a byte array.
