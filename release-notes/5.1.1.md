## New Features

- **Cross-index more-like-this sources.** More-like-this queries accept a new `sourceIndex` list so the source documents resolved from `documentId` can be fetched from indexes other than the ones the similarity search runs on. This lets you find documents in one index that are similar to a document living in a different index. When set, the queried indexes are no longer used for fetching: sources are read only from the source indexes, tried in priority order with the first index that has the document winning on collision. Wildcards and aliases resolve as usual, and source-document exclusion (`includeSourceDocs=false`) is automatically dropped when the source indexes are disjoint from the queried indexes, since those sources cannot appear in the results. Available in the Java client via `MoreLikeThisQuery.addSourceIndex(...)`.

## Improvements

- **Vector-only more-like-this now behaves like a vector query.** A `MORE_LIKE_THIS` query with only a vector field (no lexical terms) now caps the merged result set and `totalHits` at `vectorTopN` globally, matching the semantics of a plain `VECTOR` query, instead of returning up to `vectorTopN` hits per shard. Hybrid more-like-this remains uncapped because lexical matches extend beyond the KNN candidate pool. `vectorTopN` is now validated as positive and defaults to 100 when unset.

## Bug Fixes

- **Taxonomy is now committed before the main index.** The index references taxonomy ordinals, so the taxonomy must be durable first. A crash between the two commits now leaves at most a harmless taxonomy superset rather than index ordinals with no matching taxonomy entry.
- **Spreadsheet `getInt(index, defaultValue)` returned the wrong values.** A copy-paste error left this `SpreadsheetRecord` helper named `getDouble`; it is now correctly exposed as `getInt(int index, int defaultValue)`.
- **Zulia AI classifier stats and model fixes.** Fixed feature-stat persistence reading only 9 of the 10 saved doubles (the `max` value is now restored, correcting an off-by-one in the loaded stats); `categoryRecall` now returns recall instead of precision; per-category stat loops now actually run because the stats list is populated rather than only sized; and the multi-classifier seed maximum now starts at negative infinity instead of `Float.MIN_VALUE` (a small positive number).
